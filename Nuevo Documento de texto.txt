Plan: DataLens AI — Rediseño UX Multi-Archivo
Contexto
El documento multi_archivo_tecnico.docx define un cambio mayor de UX: pasar de un wizard secuencial de 6 pasos (un archivo) a una interfaz sidebar de archivos + área principal con pestañas de trazabilidad, soportando múltiples archivos con procesamiento paralelo controlado.

Principio clave: El backend (route.ts, DataStore, todos los agentes) no cambia. Ya es session-keyed, por lo que múltiples sessionId paralelos ya funcionan. El cambio es 100% en el frontend. Todos los componentes existentes se reutilizan vía props sin modificarse.

Herramientas a usar:

GSD (IMPROVERS/get-shit-done-main): Framework de desarrollo spec-driven para estructurar la ejecución por fases con agentes paralelos
UI/UX Pro Max (IMPROVERS/ui-ux-pro-max-skill-main): Base de datos de patrones de diseño — usar python3 src/ui-ux-pro-max/scripts/search.py para consultar estilos, colores, UX guidelines, antes de implementar cada fase

El tab Dashboard muestra progresivamente: si analysis === null → AnalysisPanel; si reportConfig === null → VizProposalPanel; si completo → Dashboard.

Estado Multi-Archivo (Client-Side)

// FileRecord: unidad de estado por archivo
interface FileRecord {
  fileId:          string           // uuid cliente
  fileName:        string
  sessionId:       string | null    // asignado tras clean_data
  status:          FileStatus
  rawData:         Record<string,unknown>[] | null
  cleanedData:     Record<string,unknown>[] | null
  schema:          SchemaMap | null
  schemaBlueprint: SchemaBlueprint | null
  questions:       QuestionOption[] | null
  analysis:        string | null
  analysisApproved: boolean
  vizProposals:    VizProposal[] | null
  reportConfig:    ReportConfig | null
  dataAnomalies:   DataAnomaly[]
  auditPassed:     boolean | null
  activeTab:       'original'|'schema'|'normalization'|'validation'|'dashboard'
  error:           string | null
}

// Estado global en page.tsx
Map<fileId, FileRecord>  →  gestionado por useFileQueue()
string | null            →  selectedFileId

export default function Home() {
  const queue = useFileQueue()
  const chat = useMultiChat(queue.readyFiles)
  const [showAdmin, setShowAdmin] = useState(false)
  const [chatExpanded, setChatExpanded] = useState(false)

  if (showAdmin) return <AdminView sessionId={queue.selectedFile?.sessionId} onBack={() => setShowAdmin(false)} />

  return (
    <div className="flex h-screen overflow-hidden bg-background-dark">
      <FileSidebar
        files={queue.files}
        selectedFileId={queue.selectedFileId}
        onSelectFile={queue.setSelectedFileId}
        onAddFiles={queue.enqueueFiles}
        sessionId={queue.selectedFile?.sessionId ?? null}
        onOpenAdmin={() => setShowAdmin(true)}
      />
      <main className="flex-1 flex flex-col min-w-0 relative">
        <FileMainArea file={queue.selectedFile} queue={queue} />
        <GlobalChatBar
          readyFilesCount={chat.readyFilesCount}
          chatHistory={chat.chatHistory}
          isTyping={chat.isTyping}
          onSend={chat.sendMessage}
          isExpanded={chatExpanded}
          onToggleExpand={() => setChatExpanded(v => !v)}
        />
      </main>
    </div>
  )
}

Riesgos y Mitigaciones
Riesgo	Mitigación
Map no dispara re-render en React	Siempre setFiles(new Map(prev).set(...)) — nunca mutar el Map in-place
Pipeline falla sin actualizar status	Todo runPipeline en try/catch global → siempre setea ERROR
ResponsiveContainer colapsa en tab	Wrapper del tab con h-[calc(100vh-200px)] explícito
Tailwind purga clases de status dot	Definir clases en globals.css con @apply, no strings dinámicos
AWAITING_VALIDATION ocupa slot de cola	getActiveProcessingCount excluye este estado explícitamente
